### **1. Finding Your LOCR**
**Introduction**:  
*All the text you see in-game are held inside* <Tooltip text={<strong>LOCR Files</strong>} tooltip="A file that holds game text in different languages." /><><span className="icon-alert"></span></>.  
*Let's locate one by using their 'unique identifiers' called a* <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a game file." /><><span className="icon-alert"></span></>. 
<details>

<summary>**Help:** HASH?</summary>

>Hashes are a file's <Tooltip
  text={<strong>assembly path</strong>}
  tooltip="The full file path used by the game engine to locate specific files. (filePaths + fileNames)"
/> converted into code. 

They are like shortcuts to files, making them easier to work with since they’re much shorter than full file paths.

</details>
 
####  1. Identify your LOCR
 Below are some of the <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> for the suits: 
    >- Season 1 suits: `00985A1100E5EDDC`,
    >- Season 2 suits: `009F430D046716BE` ,
    >- Season 3 suits: `004B8C5124A49543` 
2. **Copy the Hash**:  
Know when your suit was released.

3. **Go to your File**:  
Go to the <Tooltip text={<strong>game contents tab</strong>} tooltip="A tab in Glacierkit where you search game files" /> and paste your <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a game file." />
> It will lead you straight to your *LOCR File* named like  
>`randomwords.sweetmenutext].pc_localized-textlist` 


*If you need to find text for something other than suits, use the Advanced Search Guide, below*

<summary>**Advanced search guide**</summary>

   - Check only the `chunk0` box.

### **2. Get the string**

*Once you’ve opened the LOCR file, it’s time to extract the text we’ll be modifying by finding its* <Tooltip text={<strong>string</strong>} tooltip="A unique identifier for text inside a LOCR file." />.<><span className="icon-alert"></span></>


1. **Search for Your Text**: 

>Use `Ctrl + F` inside the "Preview Box" to quickly locate the text you want to modify.
:::warning Search mode
If you don’t click inside the *Preview* box first, the search won’t work properly and may look for text elsewhere in Glacierkit.
>*This feature is useful for other situations but doesn’t apply here.*
:::
2. **Copy the String**: 

>Copy **only the string** (without "quotes") attached before the text you want to change.


<details>
<summary>**Help:** String?</summary>

The <Tooltip text={<strong>string</strong>} tooltip="A unique identifier for text inside a LOCR file." /> is the long piece of code that comes *before* each entry of text you see in-game.

Example of a Suit's Name line in a LOCR File:
```json
"989928F2-06D6-42F3-871A-353F07DEF969_LEGACY_HERO_BLOODMONEYSUIT_M_HPA2293_NAME_": "Blood Money Suit"
```
- The **string**: `989928F2-06D6-42F3-871A-353F07DEF969_LEGACY_HERO_BLOODMONEYSUIT_M_HPA2293_NAME_`
- The **text**: `Blood Money Suit`
>Each text you see in game are hold by these "strings". Similar to the HASHes for game files, they are their 'unique identifiers'. Think of them as tags.
:::danger no quotes
Ensure you copy the string **without quotes**, or the process will fail later on!
>You won't even realize until you test it in-game
:::

</details>

### **3. Manifest your Text**
*Now that we have the associated string, we will* <Tooltip text={<strong>structure the manifest</strong>} tooltip="If you're curious,  
Check out the LOCR breakdown to see how it all fits together." /> *using* <Tooltip text={<strong>localisation Overrides</strong>} tooltip="A way to replace or modify in-game text, without altering the core game files." /><><span className="icon-alert"></span></>

<details>
<summary>***LOCR*** *breakdown*</summary>

Let's take a closer look at a simplified LOCR file to understand how it's set up:

```json
"en": {
   "string1": "text you see in game",
   "string2": "More text you see in game",
   "string3": "Too much text you see in game"
},
"fr": {
   "string1": "text tu vieux en jeu",
   "string2": "Plus de text tu vieux en jeu",
   "string3": "Trop de texte que vous voyez dans le jeu"
}
```

*Notice the deliberate structure and repeated elements in the LOCR file.  
This format follows a set of guidelines known as the* <Tooltip text={<strong>"Syntax"</strong>} tooltip="A set of rules that define how code is written and organized" /><><span className="icon-alert"></span></>

>*We will recreate this structure in the manifest to replace the text,  
similar to how we handled [blobsfolders](addingblobs.md#step-3-set-up-a-blobsfolder).*
<details>
<summary>More on ***Syntax***</summary>

Looking carefully at the Syntax of a LOCR file, we can learn that:
- **Unique "Strings"** are attached to the text you see in-game, separated by a `:` in each entry
- `{Language arrays}` are expressed using <Tooltip text={<strong>language codes</strong>} tooltip="The language identifiers in each language array of a LOCR file" /> (e.g., `"en"` and `"fr"`)
- Every essential part is enclosed in `"quotes"`.
- Both entries and language arrays are separated by `commas`, except for the last one.
- The **same "strings"** are reused across different language arrays  
*(Functioning as tags that map the same text.)*


</details>


</details>


1. **Open Your Manifest**:  
Navigate to your manifest file from the 'Files' Tab in GlacierKit 

2. Add this line: `"localisationOverrides": {}`  
Either copy it or type it in manually using *suggestions*

:::tip using "suggestions"
Type smarter with the <Tooltip text={<strong>suggestions feature</strong>} tooltip="A tool that helps complete and format your code efficiently as you type, reducing mistakes." /><><span className="icon-alert"></span></>. 
>*Suggestions will show up as soon as you type a **quote** `"`*

It will auto-complete most of the tedious functions in the steps to follow like:
- adding `"quotes"`, 
- adding these `:`
- adding `{brackets}`
- and more

*This feature is built-in Glacierkit or in in VS code with a* <Tooltip text={<strong>'schema'</strong>} tooltip="A guideline that defines and validates code formats ensuring error-free code" />.
:::


3. Add your <Tooltip text={<strong>LOCR HASH</strong>} tooltip="The code that appeared on top when you clicked on the LOCR File" /> and a <Tooltip text={<><strong>language bracket</strong><span className="icon-alert"></span></>} tooltip="A single language array that can contain multiple entries of strings" />: 
```json
   "localisationOverrides": {
       "yourLOCRhash": {
           "english": {}
       }
   }
   ``` 

:::danger HOLD IT
  - The next logical step would be to add our string and new text, like so:
 ```json
   "localisationOverrides": {
       "yourLOCRhash": {
           "english": {
            "string_you_copied": "Your new text"
           }
       }
   }
   ``` 
  > **But we can't do that!**
  
   *We first need to "convert" the string*
:::

### 4. **Converting the String**
*We'll have to convert the string into a* <Tooltip text={<strong>decimal number</strong>} tooltip="A string converted into a numeral form" /><><span className="icon-alert"></span></> *before using it in the manifest.*

To do that, you first have to:
1. **Convert Your String**:
    - **Open the** <Tooltip text={<strong>Text Tools tab</strong>} tooltip="A tab in GlacierKit that provides tools for converting text and hashes for game modifications." /><><span className="icon-alert"></span></>
 
    - Paste your string inside the <Tooltip text={<strong>'Localisation Hash Calculator'</strong>} tooltip= "A tool that converts your string into a format the manifest can use, generating both hex and decimal codes." /><><span className="icon-alert"></span></> box.

2. **Copy only the Decimal number**:  
Once pasting your string in the box you will see 2 codes below it.  
    >*Ignore the hex code as it's not needed*

:::tip Windows Clipboard 
Try using <Tooltip text={<strong>Windows clipboard</strong>} tooltip="A built-in Windows tool that can store multiple copied items." /> (`Windows + V`) to manage multiple copied items.  

*It’s a handy alternative to `Ctrl + V` for pasting.* 
:::

3. **Add decimal and New Text**:  
Return to your manifest and paste the <Tooltip text={<strong>new decimal</strong>} tooltip="A string converted into a numeral form" /> with your custom text.


Result:
```json
"localisationOverrides": {
    "yourLOCRhash": {
        "english": {
            "yourconvertednamedecimal": "Your new text"
        }
    }
}
````

Next up you can: 
- **Save** the manifest
- **and deploy** your mod in SMF!
>In that order

## Beyond Basics

<details>

<summary>The Description</summary>

**4. (Optionally) Change the Description**

*To replace the description of your suit as well:*

- Head back to your <Tooltip text={<strong>LOCR File</strong>} tooltip="A file that holds game text in different languages, linked by unique 'strings'." />
- Find and convert <Tooltip text={<strong>the string of suit's description</strong>} tooltip="Usually right below the string of your suit's name" /> using Text Tools
- add the new <Tooltip text={<strong>decimal</strong>} tooltip="A string converted into a numeral form" /> to your manifest. 
>Full example of a suit mod:
```json
"localisationOverrides": {
        "yourLocrHASH": {
            "english": {
                "yourconverteddecimal": "New Suit Name",
                "yourotherconverteddecimal": "New Suit Description"
            }
        }
    }
```
</details>

<details>


<summary>More language Support</summary>

**5. (Optionally) Add more languages**
#### Same Strings:
To have your mod support more languages for users, you will need to make new <Tooltip text={<><strong>language arrays</strong><span className="icon-alert"></span></>} tooltip="A single language array consisting of strings holding text" />

    

To add more languages:
- make another <Tooltip text={<strong>language bracket</strong>} tooltip="A single language array that can contain multiple entries of strings" /> 
- add the same<Tooltip text={<strong> decimal</strong>} tooltip="A string converted into a numeral form" /> with your text


Example of a new language line:
```json
"localisationOverrides": {
        "yourLocrHASH": {
            "english": {
                "yourconverteddecimal": "New Suit Name",
                "yourotherconverteddecimal": "New Suit Description"
            },
            "french": {
                "yourconverteddecimal": "Niveua Costume nome"
            }
        }
    }
```
>***Strings*** *are the same in each language array, making adding languages very easy in the manifest as well.*

:::warning language codes
- The reason why `french` is used here and not `français` is because the<Tooltip text={<strong> manifest's schema</strong>} tooltip="A structure used to validate and provide auto-completion for your manifest, helping you write error-free code." /> is coded in English. 
- As for why it doesn't use the same <Tooltip text={<strong>language codes</strong>} tooltip="The language identifiers in each language array of a LOCR file" /> as the actual <Tooltip text={<strong>LOCR Files</strong>} tooltip="A file that holds game text in different languages, linked by unique 'strings'." /> like `en` or `fr`, is because the manifest's schema is coded like that. 

To see how all languages have to be written down in the manifest, see the
    - <Tooltip text={<><strong>documentation in SMF</strong><span className="icon-alert"></span></>} tooltip="It's the book icon on the left after enabling developer mode in SMF." />. 
    - Or *expand* the tab below

:::

<details>
<summary>More LOCR files</summary>

*If you need to modify more text that are in different* <Tooltip text={<strong>LOCR Files</strong>} tooltip="A file that holds game text in different languages, linked by unique 'strings'." />: 

1. **Insert a Comma** after your previous <Tooltip text={<><strong>LOCR Array</strong><span className="icon-alert"></span></>} tooltip="an array in the manifest resembling the game's LOCR file" />  in the manifest
2. **Repeat [the steps from start](#1-finding-your-locr)** (or continue here)
2. **Add a New LOCR Array**: Add your LOCR's <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> and make new <Tooltip text={<strong>language arrays</strong>} tooltip="A single language array that can contain multiple entries of strings" /> as needed
3. **Repeat the Conversion**: Use <Tooltip text={<strong>Text Tools</strong>} tooltip="Using the 'Localisation HASH calculator'" /> again to convert any new <Tooltip text={<strong>strings</strong>} tooltip= "A key that links to text in the LOCR File" /> into <Tooltip text={<strong>decimals</strong>} tooltip="A string converted into a numeral form" />.
4. **Update the** <Tooltip text={<strong>manifest</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />: Add the new decimals and your new text.


Example of every aspect from this tutorial applied:
```json
"localisationOverrides": {
    "yourLocrHASH": {
        "english": {
            "yourconverteddecimal": "New Suit Name",
            "yourotherconverteddecimal": "New Suit Description/or Other New Suit name"
        }
    },
    "otherLocrHASH": {
        "english": {
            "somenewconverteddecimal": "More New Suit Name",
            "someotherconverteddecimal": "More New Suit Description/or Other More New Suit name"
        },
        "spanish": {
            "somenewconverteddecimal": "Más nombre de traje nuevo",
            "someotherconverteddecimal": "Más descripción del traje nuevo/u otro nombre del traje más nuevo"
        }
    }
}
```

>*Remember the* <Tooltip text={<strong>syntax rules</strong>} tooltip="A set of rules that define how code is written and organized." /> *and to keep your manifest clean and organized to avoid errors.*
