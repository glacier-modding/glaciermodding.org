## Pt. 1: **LOCATE YOUR LOCR** 
*A ***LOCR File*** is basically a "container" for the game's text you see in game.* 


### 1. Identify your LOCR
 Below are some of their <Tooltip text={<><strong>HASH</strong><span className="icon-alert"></span></>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> lines: 
- Season 1 suits: `00985A1100E5EDDC`,
- Season 2 suits: `009F430D046716BE` ,
- Season 3 suits: `004B8C5124A49543`

## Pt. 2: **GET YOUR STRING**
*Every line of code in a* <Tooltip text={<strong>LOCR File</strong>} tooltip="A file that holds game text in different languages, linked by unique 'strings'." /> *consists of a* 
>   - <Tooltip text={<><strong>string</strong><span className="icon-alert"></span></>} tooltip= "A key that links to text in the LOCR File" /> 
>   - *attached to the text you see in-game*.

Simplified version of a LOCR file
```json
     "en": {
        "string1": "text you see in game",
        "string2": "More text you see in game"
     },
     "fr": {
     "string1": "francais text tu vieux en jeu",
     "string2":"Psus text tu vieux en jeu"
     }
     ```
    [[Return to 'Syntax']](#syntax)
    
    Example of a suit's name *line*:
 ```json
 "989928F2-06D6-42F3-871A-353F07DEF969_LEGACY_HERO_BLOODMONEYSUIT_M_HPA2293_NAME_": "Blood Money Suit"
 ```
 - The first part is the "string": `989928F2-06D6-42F3-871A-353F07DEF969_LEGACY_HERO_BLOODMONEYSUIT_M_HPA2293_NAME_`
 - The second part will be the text you see in game: `Blood Money Suit`

### 2. Convert your string

- Copy *only* the converted <Tooltip text={<strong>decimal number</strong>} tooltip="A string converted into a numeral form" />. 

>*We now have everything needed for the next part:*
 >- a <Tooltip text={<strong>decimal form</strong>} tooltip="A string converted into a numeral form" /> of our <Tooltip text={<strong>string</strong>} tooltip= "A key that links to text in the LOCR File" />

## Pt. 4: **UPDATE MANIFEST**
*Here we will be adding* <Tooltip text={<><strong>localisation override code</strong><span className="icon-alert"></span></>} tooltip="A way to replace or modify in-game text, like names or descriptions, with your own custom text. It allows you to customize how text appears without altering the core game files." /> *to your manifest file. It will contain:*
- <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> of your <Tooltip text={<strong>LOCR File</strong>} tooltip="A file that holds game text in different languages, linked by unique 'strings'." />
- <Tooltip text={<strong>decimal form</strong>} tooltip="A string converted into a numeral form" /> of your <Tooltip text={<strong>string</strong>} tooltip= "A key that links to text in the LOCR File" />
- *Your new text*

### 1. Add the localisation override line 

- Open your manifest
- Add this line: `"localisationOverrides": {}`

<details>

<summary>How-to Manifest</summary>

Theres *2 methods* to doing this:
- *Method 1*: 
    - just copying: `"localisationOverrides": {}`
>Or
- *Method 2*:
    - manually typing it in, starting with `"`
    - navigate the *suggestions menu* to find `localisationOverrides`.
    

>Either method should result in:
    >  ```json
    > "localisationOverrides": {}
    >  ```
:::tip using "suggestions"
*Now you know how to use the* <Tooltip text={<strong>suggestions feauture</strong>} tooltip="This feauture is only supported in Glacierkit or in VS code with a schema." />. It will auto-complete most of the tedious functions in the steps to follow like:
- adding "quotes", 
- adding one of these `:`
- adding `{brackets}`
- adding spacing to keep code readable 
- and more

>Suggestions will show up as soon as you type a **quote** `"`
:::
</details>
### 2. Insert LOCR HASH 
- Place your <Tooltip text={<strong>LOCR HASH</strong>} tooltip="The code that appeared on top when you clicked on the LOCR File" /> inside the curly brackets in "quotes" and add another `:` and `{}` after that

Result:

   ```json
   "localisationOverrides": {
       "yourLOCRhash": {}
   }
    ```

### 3. Set your language

- Add `"english": {}` to indicate which language to override.
<div id="syntax"></div>
   Result:
   ```json
   "localisationOverrides": {
       "yourLOCRhash": {
           "english": {}
       }
   }
   ```


### 4. Add your decimal and text
- Inside the language brackets, paste your <Tooltip text={<strong>decimal number</strong>} tooltip="A string converted into a numeral form" /> in "quotes"
- Add your new text... Also in "quotes".

Result:
```json
"localisationOverrides": {
    "yourLOCRhash": {
        "english": {
            "yourconvertednamedecimal": "New Suit Name"
        }
    }
}
````

### 5. (Optionally) Add more languages
#### Same Strings:
In [Part 2](#2-copy-the-string) of the tutorial we demonstrated that *strings* are the same in each <Tooltip text={<><strong>language array</strong><span className="icon-alert"></span></>} tooltip="A single language array in a LOCR file consisting of strings" />
>*This will make adding languages for each entry very easy.*
    

To have your mod support more languages, simply:
- make another <Tooltip text={<strong>language bracket</strong>} tooltip="A single language array that can contain multiple entries of strings each attached to text you see in-game" /> 
- add the <Tooltip text={<strong>same decimal</strong>} tooltip="A string converted into a numeral form" /> with your text


Example of a new language line:
<div id="langcode"></div>
```json
"localisationOverrides": {
        "yourLocrHASH": {
            "english": {
                "yourconverteddecimal": "New Suit Name"
            },
            "french": {
                "yourconverteddecimal": "Niveua Costume nome"
            }
        }
    }
```

- As for why it doesn't use the same <Tooltip text={<strong>language codes</strong>} tooltip="The language identifiers in each language array of a LOCR file" /> as the actual <Tooltip text={<strong>LOCR Files</strong>} tooltip="A file that holds game text in different languages, linked by unique 'strings'." /> like `en` or `fr`, is because the manifest's <Tooltip text={<strong>schema</strong>} tooltip="A structure used to validate and provide auto-completion for your manifest, helping you write error-free code." /> is coded like that. 

To see how all languages have to be written down in the <Tooltip text={<strong>manifest</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />, see the <Tooltip text={<strong>manifest documentation in SMF</strong>} tooltip="It's the book icon on the left after enabling developer mode." />.

### 6. Add the description
*Now try the same for the description as well.*

- <Tooltip text={<strong>convert</strong>} tooltip="Using the Localisation Hash Calculator box" /> the <Tooltip text={<strong>string</strong>} tooltip= "A key that links to text in the LOCR File" /> from the <Tooltip text={<strong>suit's description</strong>} tooltip="Usually right below the string of your suit's name" />

## Advanced Search Guide AI

When searching for text beyond suits, the **Advanced Search Tab** in GlacierKit provides extended capabilities.

### How to Access Advanced Search:
1. **Click** the **Advanced Search Tab** (magnifying glass icon on the left).

### Setting Up Your Search:
- **Select**: Only check the `chunk0` box.
- **Filter**: Enable **"localisation"** to narrow results specifically to text files.

   :::note
   - *`Chunk0` is the primary source for text data.*
   - *Using the "localisation" filter speeds up your search.*
   :::

### Searching Effectively:
*Remember, searches are case-sensitive.* 
- **Try variations** of your text to cover all bases:
   - "Blood Money"
   - "blood money"
   - "BLOOD MONEY"
   - "bloodmoney"

   :::tip Multiple Searches
   You can run several searches at once. Watch the task bar at the bottom for updates on their progress.
   :::

### Finding the Right File:
- Look specifically for **LOCR Files**. 
- **Skip**: `Line` or `dlge` files; they arenâ€™t relevant for localisation overrides.

   :::info Reference Tip
   If you come across `Line` or `dlge` files, use the **Reference List** tab to trace them back to the correct **LOCR File**.
   :::

#### Step-by-Step Overview

1. **Locate LOCR File**: Use provided hashes or <Tooltip text={<strong>Advanced Search</strong>} tooltip="On the left with a magnifying glass icon" />.

3. **Convert to Decimal**: Use <Tooltip text={<strong>Text Tools</strong>} tooltip="Use Localisation Hash Calculator" /> to convert the string to a <Tooltip text={<strong>decimal</strong>} tooltip="numbers only code" />.
4. **Update Manifest**:
   ```json
   "localisationOverrides": {
       "yourLOCRhash": {
           "english": {
               "yourconverteddecimal": "New Suit Name",
               "yourotherconverteddecimal": "New Suit Description"
           }
       }
   }
   ```
5. **Add Translations**: Extend the language array as needed.
