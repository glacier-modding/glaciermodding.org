## Pt. 1: **LOCATE YOUR LOCR** 
*A* <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the gameâ€™s text, like names and descriptions, organized for different languages." /> *is basically a "container" for the game's text you see in game.* 


### 1. Identify your LOCR
Almost all <Tooltip text={<strong>localisation</strong>} tooltip="The process of adapting content for different languages and regions, such as translating text." /> (text you see on screen) are inside <Tooltip text={<strong>LOCR Files</strong>} tooltip="A file holding the gameâ€™s text, like names and descriptions, organized for different languages." />. Below are some of their <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> lines: 
- Season 1 suits: `00985A1100E5EDDC`,
- Season 2 suits: `009F430D046716BE` ,
- Season 3 suits: `004B8C5124A49543` 
:::info HASH
You can interpret <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> lines as a file's specific address or coordinates. They are the file's *unique identifiers*
>You will constantly have to use these throughout modding.
:::

    - named like `randomwords.sweetmenutext].pc_localized-textlist`

- <Tooltip text={<strong>Open it</strong>} tooltip="By clicking it" />

## Pt. 3: **CONVERT TO DECIMAL**
We need to convert the <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> to a <Tooltip text={<strong>decimal number</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> so we can use it in our <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />.
That's what the <Tooltip text={<strong>Text Tools</strong>} tooltip="A tab in GlacierKit that provides tools for converting text and hashes for game modifications." /> tab is for.
>Converting the <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> to a <Tooltip text={<strong>decimal</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> allows us to use it as a *"key"* in the localisation override manifest.

## Pt. 4: **UPDATE MANIFEST**
*Here we will be adding a **Localisation Override** line to your* <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information" />. *It will contain:*
- <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> of your <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the gameâ€™s text, like names and descriptions, organized for different languages." />
- <Tooltip text={<strong>decimal form</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> of your <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." />
- *Your new text*

### 1. Add the localisation override line 

- Open your <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />
- Add this line: `"localisationOverrides": {}`

Theres *2 methods* to doing this:
- *Method 1*: 
    - just copying: `"localisationOverrides": {}`
    - pasting it in your <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />

 Or

- *Method 2*:
    - manually start typing it in, starting with `"` and then continue typing
    - navigate the *suggestions menu* to find `localisationOverrides`.
    

>Either method should result in:

   ```json
   "localisationOverrides": {}
   ```
:::tip using "suggestions"
Now you know how to use the **suggestions feauture**. It will auto-complete most of the tedious functions in the following steps like:
- adding "quotes", 
- adding one of these `:`
- adding brackets
- and more

>Suggestions will show up as soon as you type a **quote** `"`
:::

### 2. Insert LOCR HASH 
- Place your <Tooltip text={<strong>LOCR HASH</strong>} tooltip="the code that appeared on top when you clicked the file" /> inside the curly brackets in "quotes" and add another `:` and `{}` after that

Result:

   ```json
   "localisationOverrides": {
       "yourLOCRhash": {}
   }
    ```

### 3. Set your language

- Add `"english": {}` to indicate which language to override.

   Result:
   ```json
   "localisationOverrides": {
       "yourLOCRhash": {
           "english": {}
       }
   }
   ```
:::info Syntax
If you're paying attention, you might notice that we're setting up the ***LocalisationOverrides*** code in the <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" /> 
just like our <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the gameâ€™s text, like names and descriptions, organized for different languages." /> 
where we got the <Tooltip text={<strong>strings</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> from. 
Both will be using the same <Tooltip text={<strong>'syntax'</strong>} tooltip="A set of rules that define how code is written and organized." />.


> *Even better too if you already have noticed how similar this method is to [the previous *blobs article*](addingblobs.md#step-3-set-up-a-blobsfolder) where we 'recreated' an assembly path.*
:::

### 4. Add your decimal and text
- Inside the language brackets, paste your <Tooltip text={<strong>decimal number</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> in "quotes"
- Add your new text... Also in "quotes".

Result:
```json
"localisationOverrides": {
    "yourLOCRhash": {
        "english": {
            "yourconvertednamedecimal": "New Suit Name"
        }
    }
}
````
